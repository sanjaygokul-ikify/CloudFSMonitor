# Contribution Guidelines

## PR Workflow
1. Fork the repository: `git fork https://github.com/your-username/CloudFSMonitor.git`
2. Create a new branch: `git checkout -b feature/your-feature`
3. Commit changes: `git commit -m "Your commit message"`
4. Push changes: `git push origin feature/your-feature`
5. Create a pull request
## Commit Standards
* Follow the GitHub commit message guidelines
* Use meaningful commit messages that describe the changes made
## Code Style Rules
* Follow PEP 8 guidelines for Python code
* Use consistent naming conventions and indentation
